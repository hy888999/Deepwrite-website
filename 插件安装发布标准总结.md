# DeepWrite 插件安装与发布标准总结（可复用）

更新时间：2026-03-02
适用范围：Obsidian 系列插件（当前已落地：DeepWrite Themes、DeepWrite Export HTML）

## 1. 目标与结论

本轮工作已统一“安装认知 + 包命名 + 网站文案 + 下载产物”四件事，后续新插件可直接复用。

核心结论：

- 统一采用**手动安装**叙事，不再依赖“运行安装程序”。
- 明确告诉用户：**文件放置过程就是安装过程本身**。
- 安装包内统一包含说明文件：`文件放置(安装）说明.md`。
- 包名、解压目录名、网站产品页标题/下载链接保持一致。

---

## 2. 用户口径（必须统一）

对外文案必须强调：

1. Obsidian 是应用程序，Vault 是笔记库目录。
2. 插件必须放在 Vault 的 `.obsidian/plugins/...` 下。
3. `.obsidian` 是隐藏目录，Finder 可按 `Cmd + Shift + .` 显示。
4. **不运行安装器**；把 `main.js` 与 `manifest.json` 放到指定目录即安装完成。

---

## 3. 安装说明文件规范

安装包内说明文件统一命名：

- `文件放置(安装）说明.md`

内容结构建议：

1. 先说结论（放置文件=安装）
2. 解释 Obsidian / Vault / 插件目录关系
3. 手动安装步骤（7~8 步）
4. 成功验证方式
5. 新手 FAQ（隐藏目录、路径放错、多 Vault）

---

## 4. 打包命名规范

命名需同时满足：压缩包名 = 产品名；解压目录名 = 产品名。

当前标准：

- `DeepWrite Themes for Obsidian.zip`
  - 解压目录：`DeepWrite Themes for Obsidian/`
- `DeepWrite Export HTML for Obsidian.zip`
  - 解压目录：`DeepWrite Export HTML for Obsidian/`

可保留版本包：`<产品名> v<version>.zip`。

---

## 5. 网站同步规范

每个产品页需同步以下字段：

- `title`：与产品正式名称一致
- `download`：指向新命名主包（URL 使用 `%20` 编码空格）
- 安装章节：必须与包内说明同口径

并在 `public/downloads/`：

- 上传主包 + 版本包 + 各自 `.sha256`
- 清理旧命名包，避免用户下载混淆

---

## 6. 发布核对清单（上线前）

1. 构建通过（插件仓库）
2. 压缩包内文件核对：
   - `main.js`
   - `manifest.json`
   - `文件放置(安装）说明.md`
   - （可选）`versions.json` / `README.md`
3. 解压目录名是否等于产品名
4. 网站产品页标题、下载链接是否一致
5. 网站下载目录是否已删除旧命名包
6. 提交并推送（插件仓库 + 网站仓库）

---

## 7. 本次已落地成果（记录）

- Themes 与 Export 均改为“手动安装 + 文件放置即安装”口径。
- 两项目安装包均包含 `文件放置(安装）说明.md`。
- 网站两个产品页均同步了命名与安装说明。
- 下载包命名、解压目录命名、页面展示命名完成对齐。

---

## 8. 后续新插件复用建议

新插件立项时，首日就建立这三样：

1. `文件放置(安装）说明.md` 模板
2. 统一打包脚本（产品名即包名/目录名）
3. 网站产品页模板（含标准安装章节 + FAQ）

这样可以在第一版上线时就避免安装理解成本与客服解释成本。
