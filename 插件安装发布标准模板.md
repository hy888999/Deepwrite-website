# <产品名> 安装与发布标准模板

更新时间：<YYYY-MM-DD>
适用范围：<插件名称，例如 DeepWrite Themes / DeepWrite Export HTML>

## 1. 目标与结论

本模板用于统一：

- 用户安装理解成本
- 安装包命名与解压目录命名
- 网站产品页标题与下载链接
- 发布核对流程

核心口径（必须一致）：

- 本插件不运行独立安装程序。
- **文件放置过程就是安装过程本身**。

---

## 2. 用户口径（对外统一）

安装说明必须解释：

1. Obsidian 是应用程序。
2. Vault 是笔记库目录（可有多个，彼此独立）。
3. 插件必须放在 Vault 的 `.obsidian/plugins/...` 目录。
4. `.obsidian` 为隐藏目录，可按 `Cmd + Shift + .` 显示。

---

## 3. 安装说明文件规范（放进安装包）

文件名统一：

- `文件放置(安装）说明.md`

内容建议结构：

1. 先说结论（放置文件=安装）
2. Obsidian / Vault / 插件目录关系
3. 手动安装步骤（流程化）
4. 安装成功验证
5. 新手 FAQ（隐藏目录、路径错误、多 Vault）

---

## 4. 打包命名规范

要求：

- 压缩包名 = 产品正式名
- 解压目录名 = 产品正式名
- 可保留版本包：`<产品正式名> v<version>.zip`

示例（按实际替换）：

- `<产品正式名>.zip`
- `<产品正式名> v<version>.zip`

---

## 5. 网站同步规范

产品页 frontmatter 至少同步：

- `title`：与产品正式名一致
- `download`：指向主包下载链接（空格需 URL 编码）

正文安装章节要求：

- 与包内 `文件放置(安装）说明.md` 口径一致
- 明确声明“文件放置即安装，不运行安装程序”

---

## 6. 发布核对清单（上线前）

1. 插件构建通过
2. 安装包内容核对
   - `main.js`
   - `manifest.json`
   - `文件放置(安装）说明.md`
   - （可选）`versions.json` / `README.md`
3. 解压目录名是否与产品正式名一致
4. 网站产品页标题、下载链接是否一致
5. 网站下载目录是否清理旧命名文件
6. 两边仓库是否提交并推送
